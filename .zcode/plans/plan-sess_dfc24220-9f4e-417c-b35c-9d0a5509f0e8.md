## 实现方案：胶囊导航指示器弹性缩放动效

### 改动范围
**仅修改 1 个文件**：`app/src/main/java/com/aiexile/animetrack/ui/components/CapsuleNavigationBar.kt`

不需要新文件、不需要新依赖、不需要改 `AnimeTrackApp.kt` 或设置项（动效是胶囊栏固有行为，无需开关）。

### 改动点 1：定义最小缩放常量（文件顶部，约第 43 行后）
```kotlin
/** 指示器滑动过程中的最小缩放比例（缩小到 60%） */
private const val INDICATOR_MIN_SCALE = 0.6f
```

### 改动点 2：计算缩放因子（`CapsuleNavigationBar` 内，约第 62 行 `effectiveIndex` 之后）
取与现有 `effectiveIndex` 同源的 pager 偏移量，无新状态：
```kotlin
// 滑动时整体缩小，到位恢复满尺寸
// abs(currentPageOffsetFraction): 静止/到位=0，滑动中→1
val indicatorScale = if (pagerState != null) {
    1f - (1f - INDICATOR_MIN_SCALE) * abs(pagerState.currentPageOffsetFraction)
} else 1f
```

### 改动点 3：给指示器 Box 加 graphicsLayer 缩放（约第 103-113 行）
在现有指示器 `Box` 的 modifier 链中插入 `graphicsLayer`，让缩放围绕指示器自身中心：
```kotlin
Box(
    modifier = Modifier
        .offset(x = indicatorOffsetDp)
        .graphicsLayer {
            scaleX = indicatorScale
            scaleY = indicatorScale
        }   // ← 新增：整体缩放，平移时缩小、到位放大
        .width(itemWidthDp)
        .fillMaxHeight()
        .clip(CircleShape)
        .background(color = MaterialTheme.colorScheme.primary.copy(alpha = 0.15f))
)
```
> 注：`offset` 在 `graphicsLayer` 外层，保证缩放只作用于高亮块本身、不影响槽位定位与平移路径；`graphicsLayer` 已在第 37 行导入，`abs` 已在第 43 行导入，无需新增 import。

### 动效原理
- `currentPageOffsetFraction` 静止时为 0 → scale = **1.0（满尺寸）**
- 滑动进行中 → |fraction| 增大 → scale 降到 **0.6（缩小）**
- 到达目标页 → fraction 回 0 → scale 弹回 **1.0（放大）**
- 视觉效果：高亮块"缩着移动 → 到位撑开"，正好对应"移动时缩小，到位置了再放大"

### 为什么用线性驱动而非 sin 曲线
`currentPageOffsetFraction` 在滑动越过半页时会发生 `currentPage` 跳变（offset 从 ~1 翻转到 ~-1）。用 `abs()` 取绝对值后该值**连续无跳变**（0→0.99→0.99→0），因此线性映射呈现干净的"缩小→平移→放大"单谷曲线。若改用 `sin(fraction·π)`，在跳变点会出现 W 形抖动瑕疵。线性最稳、最贴合语义。

### 不做的事
- 不加设置开关（动效轻量、是胶囊栏的固有质感，强行关掉反而割裂）
- 不改底部栏（BOTTOM 样式不涉及）
- 不引入独立 `Animatable` 弹性 overshoot（同步跟随已能表达需求；若日后想要"到位弹一下超出"的果冻感再叠加，避免过度设计）

### 验证
修改后用 Gradle 构建验证编译通过，并在胶囊导航模式下手动滑动确认：滑动中指示器缩小、到位放大、平移路径无偏移、深色/浅色主题均正常。
