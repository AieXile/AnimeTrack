完整实现文档：主界面卡片扇形堆叠 + 滑动切换
1. 需求总览

    保持 LazyVerticalGrid + GridCells.Fixed(columns) 的网格布局。

    将同类型多张卡片以扇形堆叠，正面一张完整卡片，其余卡片向左右两侧偏移并旋转。

    新增：支持在堆叠组件上左右滑动，使当前顶层卡片移出，底层卡片上浮成为新的顶层，实现循环切换。

    保留原有的长按选中、高亮、状态切换等交互。

2. 数据模型
kotlin

data class AnimeGroup(
    val groupKey: String,
    val items: List<AnimeItem>,          // 至少1项，顺序有意义
    val isExpanded: Boolean = false
)

    items 的顺序定义了卡片的堆叠顺序。越靠前越靠近顶层（索引0为顶层）。

    在 HomeViewModel 中将 animeList 分组为 List<AnimeGroup> 提供给网格。

3. 静态扇形布局 (基础)
3.1 参数定义
kotlin

const val MAX_VISIBLE = 5        // 最大可见层数
val offsetXStep: Dp = 8.dp       // 相邻层X偏移基值
val offsetYStep: Dp = 2.dp       // Y方向层叠厚度
val angleStep: Float = 3f        // 相邻层旋转增量(度)

3.2 偏移计算公式

给定当前顶层卡片在 items 中的索引 topIndex，对于可见卡片序列：
text

visibleIndices = [topIndex, topIndex+1, topIndex+2, ...] (循环取模)

设 visibleIndices 第0个为顶层，其余按序号从1开始分配侧边：

    i=0 → 偏移(0,0)，旋转0°

    i=1 → 左侧， level=1， offsetX = -1leveloffsetXStep, rotation = -level*angleStep

    i=2 → 右侧， level=1， offsetX = +1leveloffsetXStep, rotation = +level*angleStep

    i=3 → 左侧， level=2， offsetX = -2leveloffsetXStep, rotation = -2*angleStep

    i=4 → 右侧， level=2， offsetX = +2leveloffsetXStep, rotation = +2*angleStep

通用公式（i从1开始）：
text

side = if (i % 2 == 1) Left else Right
level = (i + 1) / 2
offsetX = (if side == Left then -level else +level) * offsetXStep
offsetY = level * offsetYStep
rotationZ = (if side == Left then -level else +level) * angleStep

3.3 静态渲染

固定 topIndex = 0 时，渲染逻辑如前述。组件整体 Box 固定为网格单元格尺寸，clipToBounds(true)，内部按底层到顶层顺序绘制，仅顶层渲染完整的 AnimeCard，其余只渲染封面缩略图并应用 graphicsLayer 变换。
4. 动态滑动切换（核心新增部分）
4.1 状态管理

在 AnimeStackCard 内部添加：

    var topIndex by remember { mutableIntStateOf(0) } —— 当前顶层卡片在 group.items 中的索引。

    滑动过程偏移量：var dragOffsetX by remember { mutableFloatStateOf(0f) } (像素)

    是否正在执行动画重置：var isAnimating by remember { mutableStateOf(false) }

4.2 手势检测

使用 Modifier.pointerInput 检测水平拖拽：
kotlin

Modifier.pointerInput(Unit) {
    detectHorizontalDragGestures(
        onDragStart = { /* 记录起始位置 */ },
        onDragEnd = { /* 判断是否触发切换 */ },
        onDragCancel = { /* 动画回弹 */ },
        onHorizontalDrag = { change, dragAmount ->
            change.consume()  // 消费事件，阻止网格滚动
            if (!isAnimating) {
                dragOffsetX += dragAmount
            }
        }
    )
}

或者使用 draggable 结合 animateTo，推荐 pointerInput 获得更精细控制。
4.3 滑动视觉反馈

    顶层卡片跟随拖拽偏移 dragOffsetX，并叠加一点点旋转（如 dragOffsetX / cardWidth * 10f 度），模拟卡片被推走。

    下层卡片根据 dragOffsetX 动态调整 graphicsLayer 的偏移与旋转，营造“下一张卡片正在浮出”的效果。

        例如，当向右滑动（拖拽值 > 0），左侧卡片（原左侧第1张）的偏移量逐渐减小，旋转趋于零，最终成为新的中心卡片。

        实现方式：在 graphicsLayer 中计算 animatedOffsetX = originalOffsetX + interpolation * dragOffsetX，animatedRotation = originalRotation + interpolation * dragOffsetX/cardWidth * ANGLE_SCALE。

    为简化，可定义两层变换：基础层为根据 topIndex 计算的静态位置 + 滑动偏移混合。预先计算 topIndex 切换后的目标偏移值，用 dragOffsetX 进度插值。

4.4 切换阈值与动画

拖拽结束时，判断 abs(dragOffsetX) > cardWidth * 0.3f（阈值可调）：

    超过阈值：

        计算目标 topIndex：向右滑 topIndex = (topIndex - 1 + items.size) % items.size；向左滑 topIndex = (topIndex + 1) % items.size。

        标记 isAnimating = true，使用 Animatable 将 dragOffsetX 动画到 0，同时将静态偏移量过渡到新 topIndex 对应的布局。

        动画结束后，更新 topIndex，清除动画状态。

    未超过阈值：动画回弹，dragOffsetX 回到 0，topIndex 不变。

动画实现技巧：使用一个 Animatable<Float> 表示 dragOffsetX 或使用 animateFloatAsState。可以将 dragOffsetX 和目标进度统一处理。一个推荐的做法是引入 progress 状态表示滑动进度（-1..1），在 graphicsLayer 中直接通过 progress 插值当前布局和目标布局。

简化方案（不插值，直接过渡）：

    滑动时实时更新 dragOffsetX 并应用到顶层和底层卡片。

    松手后，若触发切换，播放一个飞出动画：顶层卡片移到边缘消失，同时底层卡片移到中心。可使用 Animatable 控制飞出的最终位置（targetX = if (swipeRight) cardWidth else -cardWidth），底层卡片同时 animateTo 新位置。

    动画结束后重置 topIndex 并将所有偏移归零。

但考虑到代码复杂度，建议采用“滑动偏移 + 布局切换”方式：不改变 topIndex 的情况下，用 dragOffsetX 动态影响所有可见卡片的 translationX 和 rotation，松手后根据阈值，动画更新 topIndex，并伴随一个平滑过渡。过渡可使用 animateIntAsState 驱动 topIndex，但 topIndex 是离散值，无法直接插值。所以我们需要引入一个浮点型 virtualTopIndex。

推荐核心机制：

    维护 virtualTopIndex: Float（由 Animatable 驱动），初始值 0f。

    滑动时，virtualTopIndex += dragAmount / cardWidth。限制在一定范围（如 -1..1）。

    在 onDragEnd，判断 virtualTopIndex 是否需要进位/退位：

        若 virtualTopIndex > 0.3，动画到 1f，动画结束后复位：topIndex = (topIndex + 1) % size，virtualTopIndex 重置为 0f。

        若 virtualTopIndex < -0.3，动画到 -1f，结束后 topIndex = (topIndex - 1 + size) % size，重置为 0f。

        否则，动画回弹到 0f。

    布局计算：使用 effectiveTopIndex = topIndex + virtualTopIndex。根据 effectiveTopIndex 计算每张卡片的偏移，即切片索引 adjustedIndex = index - effectiveTopIndex（取模得到相对位置），再套用扇形公式。这样滑动时所有卡片位置连续变化，且松手后能平滑过渡到新布局。

公式示例：
text

fun getRelativePosition(index: Int, effectiveTop: Float): Float {
    val diff = (index - effectiveTop) % items.size
    // diff 就是相对于虚拟顶层的序号，0表示中心
    ...
}

使用 diff 替代原来的固定序号 i，即可让所有卡片随滑动移动。当 effectiveTop 接近整数时，顶层切换自然发生。
4.5 循环逻辑

组内卡片支持无限循环切换：topIndex = ((topIndex + delta) % items.size + items.size) % items.size。扇形最多可见层数仍为 MAX_VISIBLE，但循环时总能找到合适的卡片渲染。
4.6 事件冲突处理

    滑动开始后，必须消费事件，阻止父级 LazyVerticalGrid 垂直滚动。可在 onDragStart 中调用 requestDisallowInterceptTouchEvent(true)，结束或取消时恢复。

    如果卡片自身有点击事件（如状态按钮），需保证按钮在拖拽距离很小时能正常响应。可在 detectHorizontalDragGestures 中配合 detectTapGestures，或使用 pointerInput 同时处理 tap 和 drag，通过判断 dragOffsetX 是否小于触摸 slop 来决定是点击还是拖拽。

5. 动画进场与整合

    网格项整体的 animateEnter 仍应用于 AnimeStackCard 最外层。

    切换动画（内部滑动）不应影响进场动画，二者独立。

6. 性能与优化

    virtualTopIndex 变化会导致大量重组，请将 graphicsLayer 参数化，减少组合过程；使用 derivedStateOf 或 remember 缓存每个卡片的变换数值。

    图片加载：下层卡片始终以缩略图显示，且仅加载 MAX_VISIBLE 张。切换 topIndex 后，之前的顶层卡片滑到底层，应复用已加载的缩略图。

    若 items 数量很多，循环切片计算需高效，可提前生成当前可见的索引环。

7. 交互细节补充

    长按选中：长按事件依然绑定在顶层卡片上，但堆叠组件整体可识别长按。当手指按下时，若没有水平移动（或移动小于阈值），则触发长按，选中当前顶层卡片。

    状态切换按钮：位于顶层卡片上，仍然可用。滑动过程中禁用按钮。

    展开查看全部：点击 +N 标签或双击堆叠区域，可通过弹窗/底部抽屉展示该组全部卡片列表。此交互与滑动不冲突。

8. 与现有网格集成步骤 (更新)

    修改数据源为 List<AnimeGroup>。

    网格项根据 items.size 决定渲染 AnimeCard 还是 AnimeStackCard。

    AnimeStackCard 内部实现上述滑动切换逻辑。

    确保 LazyVerticalGrid 的 key 使用 group.groupKey，防止滑动导致项复用错乱。

    处理触摸冲突：堆叠组件拦截水平拖拽，垂直拖拽仍交给网格滚动。

    适配不同屏幕密度，动态计算 cardWidth 和阈值。

9. 可配置参数

    fanMaxVisible：最多可见层数（默认5）

    fanOffsetXStep, fanOffsetYStep, fanAngleStep：扇形形状

    swipeThresholdRatio：切换阈值占卡片宽度的比例（默认0.3）

    swipeAnimationSpec：切换与回弹动画规格（如 spring() 或 tween(300)）

10. 注意事项

    扇形旋转可能使卡片角落超出裁剪范围，若需避免生硬裁切，可给堆叠容器增加内边距，但会导致网格单元格变大。建议适度减小 angleStep 或使用 clipToBounds = false 并让父级（网格）裁剪。
